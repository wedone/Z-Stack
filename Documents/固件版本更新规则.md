# 固件版本更新规则

## 版本号定义

固件版本通过两个 ZCL 属性标识，定义在 `zcl_samplesw_data.c`：

| 属性 | ZCL Cluster | ZCL Attribute | 变量名 | 格式 |
|------|-------------|---------------|--------|------|
| SwBuildId | genBasic (0x0000) | ATTRID_BASIC_SW_BUILD_ID (0x4000) | `zclSampleSw_SwBuildId` | `vX.Y.Z` |
| DateCode | genBasic (0x0000) | ATTRID_BASIC_DATE_CODE (0x0005) | `zclSampleSw_DateCode` | `YYYYMMDD` |

z2m 可通过读取 genBasic Cluster 查看固件版本和编译日期。

## 版本号规则

采用语义化版本 `v主.次.修订`：

| 版本段 | 何时递增 | 示例 |
|--------|----------|------|
| **主版本** (X) | 重大架构变更、不兼容修改 | v0→v1: 从 SampleSwitch 改为自定义架构 |
| **次版本** (Y) | 新增功能、向后兼容 | v0.1→v0.2: 新增断电记忆功能 |
| **修订号** (Z) | BUG修复、小调整 | v0.1.0→v0.1.1: 修复触摸检测BUG |

> v0.x.x 表示项目处于初始开发阶段，功能尚未完全稳定。

## DateCode 规则

- 格式：`YYYYMMDD`，8字节字符串
- **必须**对应实际编译/发版日期，不得沿用框架默认值
- 同一天多次编译可共用同一 DateCode
- 每次发版（非调试编译）时更新

## 发版更新清单

每次发版（版本号变更）时，**必须**同步更新以下文件：

| 序号 | 文件 | 更新内容 |
|------|------|----------|
| 1 | `zcl_samplesw_data.c` | `zclSampleSw_SwBuildId` 改为新版本号 |
| 2 | `zcl_samplesw_data.c` | `zclSampleSw_DateCode` 改为当日日期 |
| 3 | `CHANGELOG.md` | 添加新版本条目（Added/Fixed/Changed） |
| 4 | `项目进展.md` | 更新"当前状态"中的版本号 |

### CHANGELOG.md 条目格式

```markdown
## vX.Y.Z - YYYY-MM-DD

简述本版本主要内容。

### Added
- 新功能描述 (commit hash)

### Fixed
- 修复的BUG描述 (commit hash)

### Changed
- 变更描述 (commit hash)
```

## ZCL 字符串格式

ZCL 字符串以**长度前缀字节**开头，后跟字符数据，长度必须等于实际字符数：

```c
// 正确: 长度=8, 8个字符
const uint8 zclSampleSw_DateCode[] = { 8, '2','0','2','6','0','7','2','4' };

// 正确: 长度=6, 6个字符
const uint8 zclSampleSw_SwBuildId[] = { 6, 'v','0','.','1','.','0' };

// 错误: 长度与字符数不符
const uint8 zclSampleSw_DateCode[] = { 16, '2','0','2','6','0','7','2','4',' ',' ',' ',' ',' ',' ',' ',' ' };
```

> DIYRuZ_RT 原始代码用空格填充到16字节，导致 z2m 显示 `20260724        `（带尾部空格）。本项目已修正为精确长度。
