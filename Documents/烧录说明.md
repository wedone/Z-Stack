# 烧录说明

## CCLoader 烧录器

| 项 | 值 |
|----|-----|
| 硬件 | NodeMCU ESP8266 + CC2530 (CC Debug 接线) |
| 地址 | `http://10.0.0.147` |
| API 端口 | 80 (HTTP) / 81 (SSE 实时事件) |
| 烧录接口 | CC Debug (D1=RESET, D2=DC, D6=DD) |
| 串口监控 | ESP8266 RX ← CC2530 P0_3 (UART0 TX) |
| 固件格式 | **仅接受 .bin**，.hex 会被拒绝 |

> 完整 API 文档: `curl.exe -s http://10.0.0.147/api/help`

## HEX 转 BIN

CCLoader API 仅接受 `.bin`，编译产出的 `.hex` 需先转换。

工具路径: [tools/hex2bin.py](../tools/hex2bin.py)

```powershell
# 单个固件转换（默认输出同名 .bin）
python d:\vc\Z-Stack\tools\hex2bin.py "d:\vc\Z-Stack\Projects\zstack\HomeAutomation\SampleSwitch\CC2530DB\RouterEB\Exe\SampleSwitch.hex"

# 指定输出路径
python d:\vc\Z-Stack\tools\hex2bin.py input.hex output.bin

# OTA分体固件合并 (Bootloader + 应用)
python d:\vc\Z-Stack\tools\hex2bin.py Boot.hex App.hex merged.bin
```

> 默认填充到 256KB (0x40000)，缺失地址填 0xFF，适配 CC2530F256。

## 烧录流程（3步）

### 1. 上传 BIN 到 CCLoader

```powershell
curl.exe -s -F "file=@SampleSwitch.bin" http://10.0.0.147/api/upload
# 返回: {"success":true,"filename":"SampleSwitch.bin","size":262144}
```

> 注意: PowerShell 中必须用 `curl.exe`，不能用 `curl`（后者是 Invoke-WebRequest 别名）。

### 2. 发起烧录

```powershell
curl.exe -s -X POST "http://10.0.0.147/api/burn" -H "Content-Type: application/json" -d '{\"filename\":\"SampleSwitch.bin\"}'
# 返回: {"success":true,"async":true,"task_id":1,"total_blocks":512}
```

> 烧录强制异步 + 强制校验，立即返回 task_id，后台执行。

### 3. 轮询状态

```powershell
# 每2秒轮询一次
curl.exe -s http://10.0.0.147/api/status
# 烧录中: {"state":"burning","burn":{"percent":50,"current_block":256,"total_blocks":512,"done":false,"error":""}}
# 完成:   {"state":"idle","burn":{"percent":100,"done":true,"error":""}}
```

## 烧录后操作

```powershell
# 复位 CC2530
curl.exe -s -X POST http://10.0.0.147/api/reset

# 开始串口监控 (115200 baud)
curl.exe -s -X POST http://10.0.0.147/api/monitor -H "Content-Type: application/json" -d '{\"baud\":115200}'

# 获取串口日志（断点续传，since=时间戳）
curl.exe -s "http://10.0.0.147/api/monitor/buffer?since=0"

# 停止监控
curl.exe -s -X POST http://10.0.0.147/api/stop
```

## API 端点速查

| 方法 | 路径 | 功能 |
|------|------|------|
| GET | /api/help | 帮助文档 |
| GET | /api/status | 当前状态（烧录进度/监控/时间） |
| GET | /api/files | 列出已上传 BIN |
| POST | /api/upload | 上传 BIN（multipart, 字段 file） |
| DELETE | /api/files/{name} | 删除指定 BIN |
| POST | /api/burn | 烧录（强制异步+校验） |
| POST | /api/monitor | 开始监控 |
| GET | /api/monitor/buffer?since=N | 获取日志（断点续传） |
| POST | /api/stop | 停止监控 |
| POST | /api/reset | 复位 CC2530 |
| POST | /api/reboot | 重启 ESP8266 |

## 常见问题

### Q: 上传 .hex 报错 hex_not_supported？

API 仅接受 .bin，需先用 hex2bin.py 转换。浏览器端上传 .hex 会自动转换但 API 不会。

### Q: 烧录失败？

1. 检查 D1/D2/D6 三根线是否接对
2. 确认 CC2530 GND 与 NodeMCU 共地
3. CC2530 需通电（3.3V）
4. 查看 `/api/status` 的 `burn.error` 字段

### Q: PowerShell curl 报错？

PowerShell 的 `curl` 是 `Invoke-WebRequest` 别名，必须用 `curl.exe` 调用真正的 curl。
