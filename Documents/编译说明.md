## 编译配置 (Configuration)

| 配置名 | 用途 | 是否使用 |
|--------|------|----------|
| CoordinatorEB | 协调器固件 | 否 |

## 关键预定义宏

以下宏在 IAR 工程配置中定义，影响编译行为：

| 宏 | 配置 | 说明 |
|----|------|------|
| `ZCL_REPORTING_DEVICE` | RouterEB | 启用 ZCL 属性上报功能（`zcl_SendReportCmd` 可用） |
| `OTA_CLIENT` | RouterEB - OTAClient | 启用 OTA 客户端功能 |
| `ZG_BUILD_RTR_TYPE` | RouterEB | 设备类型为 Router |
| `CC2530` | 所有配置 | 目标芯片 |

## 常见编译错误

### 1. `Function "zcl_SendReportCmd" declared implicitly`

**原因**: 缺少 `ZCL_REPORTING_DEVICE` 宏定义

**修复**: 在 `SampleSwitch.ewp` 对应 configuration 的 `CCDefines` 中添加 `<state>ZCL_REPORTING_DEVICE</state>`

### 2. `Fatal error: Configuration "RouterEB - OTAClient" not found`

**原因**: 配置名包含空格和特殊字符

**修复**: 在 CLI 中用引号包裹配置名，如 `"RouterEB - OTAClient"`

### 3. OtaConverter 后构建错误

**原因**: OTA 后构建步骤调用 `OtaConverter.exe`，该工具可能不存在

**修复**: 在 IAR 工程 Options → Build Actions → Post-build command line 中清空命令，或确保 `OtaConverter.exe` 存在于指定路径

## 固件版本号

固件版本号定义在 `zcl_samplesw_data.c` 中：

```c
const uint8 zclSampleSw_DateCode[] = { 8, '2','0','2','6','0','7','2','4' };  // 编译日期
const uint8 zclSampleSw_SwBuildId[] = { 6, 'v','0','.','1','.','0' };        // 固件版本
```

**修改版本号时需同步更新**：
1. `zcl_samplesw_data.c` 中的 `zclSampleSw_SwBuildId` 和 `zclSampleSw_DateCode`
2. `CHANGELOG.md` 添加新版本条目
3. `项目进展.md` 更新当前版本号

z2m 可通过读取 genBasic Cluster 的 SwBuildId 属性查看固件版本。
