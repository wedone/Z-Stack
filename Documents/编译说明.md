# 编译说明

## 编译环境

| 工具 | 说明 |
|------|------|
| IAR Embedded Workbench for 8051 | 版本 8.10 或更高 |
| Z-Stack 3.0.2 | 已集成在项目仓库中 |
| Python 3 | 运行 hex2bin.py 转换工具 |

## IAR 安装路径

默认安装路径（如不同请替换）：

```
C:\Program Files (x86)\IAR Systems\Embedded Workbench\common\bin\IarBuild.exe
```

> 注意：路径中**不含版本号**，直接是 `Embedded Workbench\common\bin`。

## 工程文件

```
Projects/zstack/HomeAutomation/SampleSwitch/CC2530DB/SampleSwitch.eww
```

## 编译配置 (Configuration)

| 配置名 | 用途 | 当前状态 |
|--------|------|----------|
| CoordinatorEB | 协调器固件 | 不使用 |
| **RouterEB** | 路由器固件（无OTA） | **当前使用** |
| EndDeviceEB | 终端设备固件 | 不使用 |
| RouterEB - OTAClient | 路由器+OTA客户端 | 备用 |

## CLI 编译命令

```powershell
# 设置 IAR 路径（按实际安装路径调整）
$iar = "C:\Program Files (x86)\IAR Systems\Embedded Workbench\common\bin\IarBuild.exe"

# 编译 RouterEB 配置（非OTA，当前使用）
& $iar "d:\vc\Z-Stack\Projects\zstack\HomeAutomation\SampleSwitch\CC2530DB\SampleSwitch.ewp" -build "RouterEB"

# 编译 RouterEB - OTAClient 配置（OTA版，备用）
& $iar "d:\vc\Z-Stack\Projects\zstack\HomeAutomation\SampleSwitch\CC2530DB\SampleSwitch.ewp" -build "RouterEB - OTAClient"
```

## 关键预定义宏

以下宏在 IAR 工程配置中定义，影响编译行为：

| 宏 | 配置 | 说明 |
|----|------|------|
| `ZCL_REPORTING_DEVICE` | 所有配置 | 启用 ZCL 属性上报功能（`zcl_SendReportCmd` 可用） |
| `OTA_CLIENT` | RouterEB - OTAClient | 启用 OTA 客户端功能 |
| `ZG_BUILD_RTR_TYPE` | RouterEB | 设备类型为 Router |
| `CC2530` | 所有配置 | 目标芯片 |

## 常见编译错误

### 1. `Function "zcl_SendReportCmd" declared implicitly`

**原因**: 缺少 `ZCL_REPORTING_DEVICE` 宏定义

**修复**: 在 `SampleSwitch.ewp` 对应 configuration 的 `CCDefines` 中添加 `<state>ZCL_REPORTING_DEVICE</state>`

### 2. `Fatal error: Configuration "RouterEB - OTAClient" not found`

**原因**: 配置名包含空格和特殊字符

**修复**: 在 CLI 中用引号包裹配置名，如 `"RouterEB - OTAClient"`

### 3. OtaConverter 后构建错误

**原因**: OTA 后构建步骤调用 `OtaConverter.exe`，该工具可能不存在

**修复**: 在 IAR 工程 Options → Build Actions → Post-build command line 中清空命令，或确保 `OtaConverter.exe` 存在于指定路径

### 4. 编译成功但无 .hex 文件输出

**原因**: IAR 工程配置的 ExtraOutputFormat 设置错误（如输出 .sim 而非 .hex）

**修复**: 在 `SampleSwitch.ewp` 中将对应配置的 `ExtraOutputFile` 设为 `SampleSwitch.hex`，`ExtraOutputFormat` 设为 `23`（intel-extended）

## 固件版本号

固件版本号定义在 `zcl_samplesw_data.c` 中：

```c
const uint8 zclSampleSw_DateCode[] = { 8, '2','0','2','6','0','7','2','4' };  // 编译日期
const uint8 zclSampleSw_SwBuildId[] = { 6, 'v','0','.','1','.','0' };        // 固件版本
```

> 详细的版本号规则、DateCode规则、发版更新清单见 [固件版本更新规则.md](固件版本更新规则.md)。

z2m 可通过读取 genBasic Cluster 的 SwBuildId 属性查看固件版本。
